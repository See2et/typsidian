# Typsidian

Typst live preview plugin for Obsidian.

## Development loop (debug vault inside repo)

This repository includes a dedicated development vault at `debug-vault/`.

### Why this setup

- Fast feedback loop: save code -> rebuild -> plugin reload in Obsidian.
- Reproducible environment: everyone uses the same vault layout.
- Safe separation: no personal notes mixed into plugin development.

## Prerequisites

- Node.js 20+
- Git
- Obsidian desktop app

## Quick start

```bash
npm install
npm run setup:vault
npm run dev
```

Then:
1. Open `debug-vault/` in Obsidian.
2. Go to `Settings -> Community plugins`.
3. Disable restricted mode if needed.
4. Enable `Typsidian`.

## Optional: automatic hot reload

Install and register the community dev plugin Hot Reload:

```bash
npm run setup:hot-reload
```

What this does:
- Clones `pjeby/hot-reload` into `debug-vault/.obsidian/plugins/hot-reload`.
- Adds `hot-reload` into `debug-vault/.obsidian/community-plugins.json`.

After enabling Hot Reload in Obsidian, file updates to `main.js`/`styles.css` trigger automatic plugin reload.

## Scripts

- `npm run dev`: watch mode build (`src/main.ts -> main.js`)
- `npm run build`: typecheck + one-shot build
- `npm run typecheck`: strict TypeScript check
- `npm run setup:vault`: create debug vault config and symlink plugin
- `npm run setup:hot-reload`: install/register Hot Reload plugin for debug vault

## Common pitfalls

- Plugin not visible: check folder name matches `manifest.json` id (`typsidian`).
- Changes not reflected: ensure `npm run dev` is running and plugin is enabled.
- Hot reload not working: ensure restricted mode is off and `hot-reload` plugin is enabled.
