# 製品概要

Typsidian は Obsidian 上で `.typ` ファイルを扱うための軽量なプラグインで、編集中に Markdown と同等の閲覧/管理体験を狙った最小機能を提供します。

## 対象ユーザー
- Obsidian を日常利用し、Typst ファイルをノートワークフロー内で扱いたい人。
- プロトタイプ段階の機能追加を素早く検証したい開発者。

## コアな価値
- `.typ` を Obsidian 内で扱うための基本導線（拡張子登録、コンテキストメニュー、イベント連携）を提供。
- 開発ループをローカルで再現しやすい構成（`debug-vault` と `npm run dev`）により、高速検証が可能。

## 現在の主要利用シナリオ（3-5）
- `.typ` ファイルの作成（`New Typst`）を Obsidian のフォルダコンテキストから実行する。
- `.typ` が対象ファイルの開閉時に、既存の Markdown 表示レイヤーに合わせた切り替えを維持する。
- vault 内で `.typ` の create/rename/delete を観測し、操作履歴をログで把握する。

## 成功指標
- 単一ファイルでプラグイン全体の主要動作が成立していること。
- ビルド/型検査が通り、`npm run dev`/`npm run build` の再現性が担保されること。
- 開発者が debug vault を使って再現的に検証できること。

## Non-goal
- 今回の初期段階では、プレビュー描画エンジンの高度な描画最適化や全機能の完成度向上は対象外。
